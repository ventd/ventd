## v1.3.0 — Confidence-gated smart control goes live (R11)

This release completes the **convergence track**: ventd's learned ("smart") predictive
controller is now governed by two real safety signals it previously stubbed out, so
predictive control engages only when it's trustworthy and yields to the plain reactive
curve the moment it isn't. It also lands the final pieces of a top-to-bottom architecture
audit (R1–R11) and fixes two first-run papercuts.

Validated end-to-end on real hardware (13900K + RTX 4090, NCT6687D) from a clean
factory-reset install.

### Smart mode — the `w_pred_system` global gate

Predictive blending now drops to reactive the moment it shouldn't be trusted — composed
from real daemon signals instead of a hard-coded `true`. The gate closes when:

- persisted state hasn't loaded, the host is **on battery**, **inside a container**,
  running a **disk scrub**, or within the **boot / resume warm-up**;
- the install wizard didn't resolve to control mode;
- **several fans are stalled at once** — a new system-wide mass-stall detector (≥ 2 fans
  commanded to spin but reading 0 RPM within 3 minutes; a single dead or parked fan never
  trips it);
- or you turn smart mode off with the new **Settings → Smart mode → Disable smart mode**
  toggle.

The dashboard's "refused" pill now reflects this, the reason is shown in the confidence
popover, and a new `w_pred_gate` doctor check reports the verdict (benign reasons read as
OK; a mass-stall reads as a warning).

### Smart mode — per-layer drift detection

Each confidence layer (A / B / C) is now monitored by a residual **EWMA control chart**.
When a layer's model diverges from reality — a re-cabled fan, dust build-up, an ambient
shift, a fan-curve edit — its confidence decays and the controller rides smoothly back to
reactive while the model re-learns, then recovers automatically with hysteresis so it
never flickers. Conservative, HIL-tunable thresholds; surfaced per layer (residual vs.
control limit) in the dashboard confidence popover.

### Fixes

- **The installer no longer false-warns "NVML verification failed → run `usermod`" on a
  working GPU.** The post-install check now probes NVML as the daemon's actual service
  user (root on the shipped unit), instead of a hard-coded `sudo -u ventd` that the root
  daemon never uses. (#1401, follow-up to #461)
- **Settings → Calibration opens correctly.** The settings sidebar's section list had
  drifted from the page and bounced the Calibration tab back to Display; it's now derived
  from the nav links so it can't drift again. (#1398)

### Under the hood — architecture audit (R1–R11)

A senior-engineer audit landed as a sequence of behaviour-preserving refactors:
de-duplicated controller-spawn wiring, ~2.7k lines of dead / false-coverage setup code
deleted, a keyed `web.Deps` + boot-time wiring guard, a generic HAL `StateFrom`, hot-path
de-duplication in the control tick, smart-mode math moved out of `package main` into
testable `internal/` packages, per-channel locking + a cached K-cal read in the blended
controller, a uniform JSON error envelope + declarative method gates across the web API, a
split `config.go`, and per-backend HAL `Enumerate` isolation. R11 (this release's
smart-mode work) is the convergence-track finale.

### Upgrade notes

Fully backward-compatible — no config changes required. Smart-mode predictive control
activates once the daemon starts with controllable channels configured; until a layer
converges it runs reactive, exactly as before. New optional `smart.disabled` config key
(default off).
