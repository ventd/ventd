## v0.5.7 — Layer-B thermal coupling estimator

The seventh smart-mode behaviour patch. Ships the per-channel RLS
coupling estimator that learns the linear thermal coupling matrix
`b_ij = ΔT_i / Δpwm_j` from the v0.5.4 observation log. v0.5.8
(Layer C marginal-benefit), v0.5.9 (confidence-gated controller),
and v0.5.10 (doctor recovery surface) all consume the snapshot
this lands.

Implements the spec at
[`specs/spec-v0_5_7-thermal-coupling.md`](https://github.com/ventd/ventd/blob/v0.5.7/specs/spec-v0_5_7-thermal-coupling.md)
across two PRs (#736 PR-A library, #738 PR-B wiring).

### How it works

Each controllable channel gets a Shard backed by a recursive least-
squares estimator with bounded covariance directional forgetting
(R12). The Shard maintains the parameter vector θ and covariance
matrix P; rank-1 Sherman-Morrison updates via gonum's
`mat.SymRankOne` keep updates O(d²) and numerically stable.

The Runtime owns one goroutine per channel (R10 §10.5 — NOT one
per shard), drives κ identifiability checks every minute, and
persists shards every minute via spec-16 KV. Hardware change
invalidation: persisted shards carry an hwmon_fingerprint
matching the live system at boot. A mismatch discards stale
shards rather than migrating.

For v0.5.7 each shard runs in the well-posed reduced-model case
(N_coupled=0, R9 §U4) — learning thermal time constant `a` and
load coefficient `c` per channel. v0.5.8 raises N_coupled to
enable cross-fan b_ij learning once Layer C is in.

### What's new

- **`internal/coupling/`** — net-new package: `Shard` (RLS with
  rank-1 Sherman-Morrison + R12 bounded covariance, lock-free
  Snapshot.Read via `atomic.Pointer`), `Window` (60-row ring
  buffer for κ identifiability detection + co-varying fan
  Pearson detection), `Runtime` (per-channel goroutine pool,
  identifiability ticker, persistence ticker), spec-16 KV
  persistence with msgpack-encoded `Bucket` carrying schema
  version + hwmon_fingerprint + θ + serialised P.
- **Three-condition warmup gate** — Snapshot remains in
  `KindWarmup` until n_samples ≥ 5·d² AND tr(P) ≤ 0.5·tr(P_0)
  AND κ ≤ 10⁴. Per R10 §10.4.
- **Three-way κ classification** — ≤10² healthy, ≤10⁴ marginal,
  >10⁴ unidentifiable. Inf and NaN classify as unidentifiable.
  Per R9 §9.4.
- **Co-varying fan detection** — signed Pearson `ρ > 0.999`
  detects fans always commanded the same PWM (Y-cable, BIOS
  PWM-share). Anti-correlated fans are not merged. Per R10 §9.4.
- **Daemon wiring** (PR-B) — `cmd/ventd/main.go`
  `buildCouplingRuntime` constructs the Runtime alongside the
  controllers. nil for monitor-only systems. ctx-scoped goroutine
  with deterministic shutdown.

### Locked design decisions

Backed by R9-R10 (346-line research doc, identifiability and
shards):

- **RLS update:** rank-1 Sherman-Morrison via gonum's
  `mat.SymRankOne`. Direct inversion of P is O(d³) and numerically
  unstable; Sherman-Morrison is O(d²) and well-behaved when the
  denominator guard fires.
- **Bounded covariance:** post-update proportional rescale clamps
  tr(P) ≤ 100. Eigenvectors preserved, only magnitudes attenuated
  (R12).
- **Forgetting factor:** λ ∈ [0.95, 0.999], default 0.99 (mid-
  range; R12 auto-tunes from there).
- **Shard dimension cap:** N_coupled ≤ 16 (R10 §10.2). Above that
  the analytical PE conditions degrade and identifiability is
  hopeless under workload-driven excitation.
- **Channel identity:** ChannelID is a free-form string for R24
  forward-compat (the chip+channel-index identity scheme can wire
  in v0.6.0 without breaking the shard map).
- **Schema versioning:** mismatch on restore discards the
  persisted shard rather than migrating. Future schema changes are
  explicit breaking bumps.
- **Restore safety net:** tr(P) clamped on load even from older
  schema versions that pre-date the in-memory clamp.

### Audit observations folded in

This release also carries forward observations from the
post-v0.5.6 audit, applied as part of PR-A:

- **Determinism replay** —
  `TestShard_DeterministicReplay` asserts identical observation
  sequences produce byte-identical θ. Guards against
  map-iteration order or time-dependent rand leaking into the
  estimator.
- **Observability** — structured slog throughout the runtime:
  warm-start vs cold-start at shard load, periodic save outcome,
  runtime started/stopped with channel count.
- **Privacy** — `TestShard_PrivacyNoUserStrings` asserts the
  Snapshot surface holds only opaque IDs and learned
  coefficients; no comm names, no PII.
