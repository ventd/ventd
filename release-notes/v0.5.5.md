## v0.5.5 — Opportunistic active probing for Layer A gaps

The fifth smart-mode behaviour patch. Ships the Layer A gap-fill
mechanism that lets ventd visit PWM bins the controller's natural
operating range never reaches — without disturbing the user.

Implements the spec at
[`specs/spec-v0_5_5-opportunistic-probing.md`](https://github.com/ventd/ventd/blob/v0.5.5/specs/spec-v0_5_5-opportunistic-probing.md)
across two PRs (#725 PR-A, #726 PR-B).

### How it works

Once a day, on a system that has been genuinely idle for ≥10 minutes
and where the user is not at the keyboard / mouse / SSH session,
ventd writes a single PWM value to one fan for ~30 seconds, observes
the response, logs it to the v0.5.4 observation stream with
`EventFlag_OPPORTUNISTIC_PROBE`, and restores the controller-managed
value. Over a few weeks every fan has a complete response curve
without the user noticing.

### What's new

- **`internal/idle/OpportunisticGate`** — wraps StartupGate's predicate
  stack with **600 s durability** (vs StartupGate's 300 s) and adds two
  signal sources: `/proc/interrupts` deltas on input-classified IRQs
  (i8042, xhci_hcd, hid, kbd/mouse) via `/sys/kernel/irq/<n>/actions`,
  and `loginctl list-sessions` parsing for `Remote=yes Active=yes
  Idle=false` sessions. Hard preconditions (battery, container, scrub,
  blocked process, boot/post-resume warmup) inherited unchanged from
  v0.5.3.
- **`internal/probe/opportunistic`** — net-new package: `Detector`
  (gap detection over the v0.5.4 log), `Scheduler` (60 s tick goroutine,
  picks lowest-PWM gap on largest-gap channel), `Prober` (writes one
  PWM for 30 s, restores on every exit path), `install_marker` (24 h
  first-probe delay), `runtime` helpers (sysfs read/write functions
  for production), `KVLastProbeStore` (per-channel last-probe
  timestamps in spec-16 KV).
- **Schema bump 1 → 2** in `internal/observation` to register
  `EventFlag_OPPORTUNISTIC_PROBE = 1 << 13`. Reader accepts v1 and v2
  files transparently; writer emits v2.
- **Settings → Smart mode**: new "Never actively probe after install"
  toggle (default off — probing enabled). Persisted via PUT
  `/api/v1/config`. Per spec-12 amendment §3.5 / RULE-UI-SMART-07.
- **Dashboard probe-in-flight pill**: amber pulse in the topbar
  while a probe runs, with the gap PWM value rendered. Polls
  `/api/v1/probe/opportunistic/status` every 5 s.
- **`internal/web` endpoint** — `GET /api/v1/probe/opportunistic/status`
  returns the live scheduler `Status` struct.
- **Daemon wiring** — scheduler launches as a goroutine alongside the
  controllers, tracked by the daemon's `WaitGroup`, cancelled on
  shutdown. Skipped automatically in monitor-only mode (no
  controllable channels).

### Locked design decisions

Backed by R-bundle research and v0.5.5 prep:

- **Strict serial probing.** One probe in flight system-wide;
  cross-channel parallelism deferred until v0.5.7's Layer B coupling
  map exists.
- **Probe grid**: 8 raw PWM units in PWM 0–96, 16 raw units in
  PWM 97–255. Stall-PWM and min-spin always anchored when in a gap.
- **7-day per-bin cool-down** matching R10 shard TTL and R12 conf_A
  τ_recency. Aborted opportunistic probes do NOT count as visits —
  bin remains eligible for retry.
- **24-hour first-install delay** per spec-smart-mode §6.4 ("kicks in
  after 24 h"). Marker file at `/var/lib/ventd/.first-install-ts`.
- **Abort thresholds reused from v0.5.3** (`envelope.LookupThresholds(class)`).
  No separate threshold table — same per-class dT/dt and T_abs
  ceilings the envelope C/D probe uses.

### Invariants

18 new `RULE-OPP-*` rules in
[`.claude/rules/opportunistic.md`](https://github.com/ventd/ventd/blob/v0.5.5/.claude/rules/opportunistic.md),
each bound 1:1 to a synthetic subtest:

- 12× `RULE-OPP-PROBE-*` — gate firing, duration, concurrency,
  polarity routing, abort, cool-down, install delay, toggle, manual
  mode, restoration, record emission, grid spacing.
- 4× `RULE-OPP-IDLE-*` — durability, input-IRQ refusal, SSH refusal,
  hard-precondition inheritance.
- 2× `RULE-OPP-OBS-*` — schema version 2, bit-13 collision freedom.

22 new synthetic CI tests; full suite green on Ubuntu / Fedora /
Arch / Alpine / Ubuntu-arm64 with race detector enabled.

### What's NOT shipped

Per spec §2.2:

- Adaptive probe scheduling (fixed cadence + 7-day cool-down for v1).
- Multi-channel parallel probes (waits on Layer B v0.5.7).
- Probe duration sweeps (locked at 30 s).
- Doctor surface for "last 20 opportunistic probes" (waits on v0.5.10).
- `opportunistic_max_age_days` user-tunable (deferred until field
  telemetry says it matters).
- Controller-tick → observation log bridge (closing v0.5.4 gap):
  the obsWriter is initialised but the controller hot loop doesn't
  yet feed it. The opportunistic scheduler still works because each
  successful probe records its own bin in the log; over time
  Layer A coverage converges. A follow-up patch wires the natural
  controller writes into the log too.
