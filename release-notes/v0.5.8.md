## v0.5.8 — Layer-C marginal-benefit estimator

The eighth smart-mode behaviour patch. Ships the per-(channel,
workload-signature) RLS estimator that learns the function
`ΔT_per_+1_PWM = β_0 + β_1·load`. v0.5.9's confidence-gated
controller uses it to refuse ramps that pay full acoustic cost
for zero thermal benefit; v0.5.10 doctor surfaces saturation
diagnostics.

Implements the spec at
[`specs/spec-v0_5_8-marginal-benefit.md`](https://github.com/ventd/ventd/blob/v0.5.8/specs/spec-v0_5_8-marginal-benefit.md)
across three PRs: spec #740 (DESIGN), PR-A #741 (library), PR-B
#742 (wiring).

### How it works

Per (channel, signature) pair, an RLS shard estimates
`ΔT_per_+1_PWM` as a linear function of the workload-load proxy.
At admission time the shard seeds `β_0` from Layer-B's diagonal
coupling coefficient `b_ii` (R10 §10.7 informative-prior RLS) —
but only when signguard has confirmed the parent's polarity sign
via ≥5/7 opportunistic-probe agreements (R27).

Saturation fires via two paths:

- **Path A (predicted):** `(β_0 + β_1·load) × ΔPWM < 2 °C`.
- **Path B (observed):** 20 consecutive sub-2 °C writes (R11 §0
  locked).

During warmup the saturation flag is forced false — wrong-direction
prior guard (spec §3.7).

The OAT (one-at-a-time) gate rejects update samples while any
other channel's PWM is still moving; mitigates R28 multi-channel
aerodynamic interference contamination of per-channel β_0
estimates. Full R17 INTERFERENCE work stays at v0.6.0.

Persistence at `smart/shard-C/<channel>-<sig>.cbor` per R15 §104
(Layer-B parent at `smart/shard-B/`). Bucket carries
`EWMAResidual + InitialP + ObservedSaturationPWM` so v0.5.9's
R12 confidence formula and R16 anomaly detection can graft on
without forced schema bumps.

### What's new

- **`internal/marginal/`** — net-new package. `Shard` (RLS +
  dual-path saturation + ConfidenceComponents emission), `Runtime`
  (per-channel shard map + κ-deferred activation + LRU eviction +
  OAT gate + opportunistic-probe sign-guard integration), spec-16
  KV persistence with `hwmon_fingerprint` invalidation.
- **`internal/coupling/signguard/`** — net-new package. Consumes
  the v0.5.5 opportunistic-probe ground-truth stream (known PWM
  step + observed ΔT) and produces a 5-of-7 rolling sign-vote on
  Layer-B's `b_ii` polarity. Confirmed channels enable Layer-C
  prior seeding from Layer B; unconfirmed admit at θ = [0, 0].
- **`internal/idle.CaptureLoadAvg`** — exposed for the marginal
  Runtime's per-tick load proxy.
- **`Config.SmartMarginalBenefitDisabled`** toggle — default
  false; suppresses the marginal goroutine launch when set.
- **22 new RULE-CMB-* + 3 RULE-SGD-* bindings** (1:1 with
  subtests, enforced by `tools/rulelint`).

### Locked design decisions

Backed by the R-bundle (R9 + R10 + R11 + R7 are the design of
record):

- **Model dimension `d_C = 2`** per R10 §10.1: `φ = [1, load]`,
  `θ = [β_0, β_1]`. Lower than v0.5.7's `d_B` because Layer C
  sees no cross-channel coupling — that's Layer B's job.
- **Saturation threshold 2 °C / 20 writes** per R11 §0 (locked
  via `tools/rulelint` pinning).
- **Per-channel shard cap = 32** with R7-style weighted-LRU
  eviction (τ=14 d).
- **κ-deferred activation:** Layer-C shard NOT created when
  parent Layer-B `κ > 10⁴`; `τ_retry = 1 hour`.
- **Forgetting factor `λ = 0.99`** (R12 directional forgetting
  auto-tunes from there). `tr(P) ≤ 100` clamp.
