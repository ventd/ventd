## v0.5.6 — Workload signature learning + classification

The sixth smart-mode behaviour patch. Ships the library that turns
the running process set into a stable, opaque, privacy-safe label.
Layer C (v0.5.8), Layer B (v0.5.7), and the confidence-gated
controller (v0.5.9) all key per-(channel, signature) state on these
labels.

Implements the spec at
[`specs/spec-v0_5_6-workload-signatures.md`](https://github.com/ventd/ventd/blob/v0.5.6/specs/spec-v0_5_6-workload-signatures.md)
across two PRs (#730 PR-A, #731 PR-B). Closes the v0.5.4 obsWriter
→ controller gap as part of PR-B.

### How it works

Every 2 seconds the daemon hashes the comm names of every process
passing the contribution gate (EWMA-CPU > 5% of one core OR RSS >
256 MiB), maintains an EWMA-weighted multiset of those hashes, and
promotes the top-K=4 to an active "signature label" once the K-set
has been stable for ≥3 consecutive ticks (~6 s). The signature
label is what the controller stamps on every observation log
record.

All hashing is local (SipHash-2-4 keyed by a per-install salt at
`/var/lib/ventd/.signature_salt`, mode 0600); a leaked diag bundle
exposes only opaque hex tokens, never plaintext process names.

### What's new

- **`internal/signature/`** — net-new package: `Hasher` (SipHash-2-4
  keyed by salt), `Library` (EWMA-multiset + K-stable promotion),
  `Bucket` persistence to spec-16 KV, weighted-LRU eviction with
  τ=14 days, `MaintenanceBlocklist` reusing R5's blocklist as a
  positive-label dictionary (`maint/<canonical>`).
- **`internal/proc/walker.go`** — net-new shared process walker
  reading `/proc/PID/{comm,stat,statm}` with EWMA-CPU computation
  over a 10 s window. Reused by the idle gate and the signature
  library.
- **`internal/idle/blocklist_export.go`** — exports R5's
  maintenance-class names as a positive-label dictionary; single
  source of truth in `internal/idle`.
- **Controller→obsWriter wiring** (closes the v0.5.4 gap) — every
  successful PWM write now emits one observation record with
  `SignatureLabel` populated and `ChannelID` computed via FNV-1a
  of the PWM path.
- **Settings → Smart mode toggle** "Disable workload signature
  learning," default off (learning enabled in auto mode). Per
  spec-12 amendment §3.5 / `RULE-UI-SMART-07`.
- **R1/R3 disable inheritance** — signature learning automatically
  off in containers/VMs (Tier-2 BLOCK) and on hardware-refused
  platforms (Steam Deck).

### Locked design decisions

Backed by R7 (869-line research doc, exhaustive):

- **Hash:** SipHash-2-4 keyed by per-install 32-byte salt. Pure-Go
  via `github.com/dchest/siphash` (CC0).
- **Hash input:** `/proc/PID/comm` only — NOT exe, cmdline,
  parent-comm. R7 §Q3.
- **Top-N gate:** EWMA-CPU > 5% of one core OR RSS > 256 MiB;
  kthreads excluded. R7 §Q2.
- **Stability:** top-K=4 hashes by EWMA weight, K-stable promotion
  at M=3 consecutive ticks (~6 s) with half-life 2 s matched to
  R11's fast loop. R7 §Q4.
- **Library:** 128 buckets, weighted-LRU eviction with τ=14 days.
  R7 §Q5.
- **Persistence:** spec-16 KV under namespace `signature`,
  msgpack-encoded `Bucket` struct.
- **Maintenance class:** reserved labels `maint/<canonical>` from
  R5's existing blocklist for processes that dominate the K=4
  set. R7 §Q2(B).
