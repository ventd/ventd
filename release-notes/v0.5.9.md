## v0.5.9 — Confidence-gated controller + predictive install pipeline

The first release where ventd is **truly smart**: the daemon's hot
path runs a confidence-gated blend of the v0.4.x reactive curve
and an IMC-PI predictive controller derived from the layered
learning surfaces (Layer-A / -B / -C). When the per-channel
confidence weight `w_pred` is high, the controller plans ahead;
when any layer is uncertain (cold-start, drift, mass-stall), the
controller falls back to reactive — atomically, with a Lipschitz
clamp on the rate of weight change.

This release also rebuilds the install path from "install-and-pray"
to **predict-not-react**: a comprehensive preflight gates every
implicit precondition before the wizard touches the host, a
controlled cp + sign + depmod + modprobe sequence replaces the
opaque `make install` that previously made deferred Secure Boot
signing impossible, and every wizard phase has explicit
pre/postcondition gates with declarative cleanup on failure.

Implements the spec at
[`specs/spec-v0_5_9-confidence-controller.md`](https://github.com/ventd/ventd/blob/v0.5.9/specs/spec-v0_5_9-confidence-controller.md)
across eight PRs:

- **PR-A.1** #760 — Layer-A confidence estimator (`conf_A`)
- **PR-A.2** #802 — confidence aggregator (R12 chain: drift decay → min collapse → LPF → Lipschitz clamp)
- **PR-A.3** #803 — IMC-PI confidence-gated blended controller + 3-tier R8 fallback classifier
- **PR-A.4** #804 — smart-mode preset enum (Silent / Balanced / Performance) + per-channel setpoints
- **PR-B**   #807 — daemon wiring + status + settings UI surface
- **PR-C**   #810 — per-failure-class wizard + doctor classifier (cross-cutting `internal/recovery/`)
- **PR-D**   #811 — predictive install pipeline + comprehensive preflight + gated wizard

### How the controller blends

Per tick, per channel:

```
output = w_pred · predictive + (1 − w_pred) · reactive
```

`predictive` is the IMC-PI output from gains derived as
`K = |b_ii / (1 − a)|` (process-gain magnitude — see SPEC DIVERGENCE
note in `internal/controller/blended.go`), τ clamped to [50 s,
1800 s], `K_p = τ / (K · (λ + θ))`, `K_i = K_p / τ`. Polarity is
handled in `polarity.WritePWM` (RULE-POLARITY-05), not the PI math.

`w_pred` is computed by the aggregator from the three confidence
layers:

1. **Drift decay** — per-layer `0.5^(t / 60s)` when R16 reports drift.
2. **Min collapse** — weakest-link across `(conf_A, conf_B, conf_C)`.
3. **LPF** — single-pole, τ_w = 30 s.
4. **Lipschitz clamp** — |Δw_pred| ≤ 0.1 per 2 s tick.

Plus a **5-minute cold-start hard pin** after Envelope C completes
(R12 §Q4) and a **system AND-gate** (state-loaded / hard-preconds /
wizard outcome / no-mass-stall) that bypasses Lipschitz on flip
when any check fails.

### Three independent refusal gates protect closed-loop runs

1. **Instability guards** (RULE-CTRL-PI-05) — six conditions force
   `w_pred=0` for the channel-tick: nil/warming snapshot, `a∉(0,1)`,
   `b_ii=0`, `Kappa>1e4`, non-finite Layer-B values.
2. **Path-A refusal** (RULE-CTRL-PATH-A-01) — Layer-C predicts
   `|ΔT| < 2 °C` for the candidate ΔPWM → fall through to reactive
   and freeze the integrator.
3. **Cost gate** (RULE-CTRL-COST-02) — `cost(ΔPWM) > benefit(ΔPWM)`
   per the preset's k-factor → refuse the ramp.

A **first-contact clamp** prevents the predictive arm from ever
reducing cooling on a channel's first `w_pred>0` tick of its
lifetime — protects against a stale or miscalibrated estimate
driving the fan down on first engage. Re-armed only on
`probe.WipeNamespaces`.

### Wizard recovery + comprehensive preflight (PR-C + PR-D)

The wizard's calibration error banner now surfaces actionable
remediation cards (Apply fix / Show instructions / Docs only)
per failure class — Secure Boot, missing headers, DKMS build
failed, AppArmor denied, missing module. The classifier in
`internal/recovery/` is **cross-cutting**: doctor's runtime
detector loop in v0.5.10 will reuse the same `FailureClass` enum
+ `Remediation` catalogue, so adding a new failure mode means
one new entry, both surfaces pick it up.

PR-D reshapes the install path:

- **9 new `FailureClass` values** for runtime regressions:
  `MissingBuildTools`, `DKMSStateCollision`, `InTreeConflict`,
  `Containerised`, `PackageManagerBusy`, `DaemonNotRoot`,
  `ReadOnlyRootfs`, `DiskFull`, `ConcurrentInstall`.
- **Comprehensive `PreflightOOT`** — 12 new probe seams covering
  build tools, Secure Boot prerequisite chain
  (`sign-file → mokutil → MOK key`), `/lib/modules` writeability,
  container detection, apt lock, sudo / root, stale DKMS,
  in-tree driver conflicts, concurrent wizard, per-path disk space.
- **Controlled install pipeline** — 9-step sequence
  (Build → SignBuildDir? → Copy → SignInstalled? → Depmod →
  DKMS → Modprobe → Persist → Verify) replaces `make install`.
  Each step has explicit error handling and orphan-state cleanup.
- **Module signing** via kmod sign-file with MOK key discovery
  (shim-signed / `/var/lib/ventd/mok` / `/etc/ventd/mok`). Doctor's
  runtime detector inherits the same Probes machinery in v0.5.10.
- **Wizard PhaseGate machinery** with explicit pre/body/post/cleanup
  hooks and panic recovery. Wizard run lock prevents concurrent
  installs from racing each other's modprobe / DKMS state.
- **26 new RULE bindings** (preflight-comprehensive / install-pipeline /
  wizard-gates) all 1:1 with subtests and verified by `tools/rulelint`.

monitor-only catalog entries with `capability: ro_unsupported`.

### Upgrade

`apt upgrade ventd` / `dnf upgrade ventd` / `pacman -Syu` as
usual. The daemon picks up the smart-mode preset (default:
`balanced`) on next start; existing calibration data carries
forward via the cold-start hard-pin window so the controller
runs reactive-only for the first 5 minutes after a kernel
update or daemon restart.
