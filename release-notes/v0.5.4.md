## v0.5.4 — passive observation log + first-time-user polish

The fourth smart-mode behaviour patch. Ships the passive observation log
(the v0.5.4 spec) plus the page-level UI redesign and a deep first-time-
user bug-squash, validated end-to-end on a freshly-imaged Ubuntu 24.04
VM with AppArmor enforcing.

### What's new

- **Passive observation log** (`internal/observation`, #693). Per-tick
  controller records (PWM written, RPM observed, sensor readings,
  classifier state, event flags) are appended to a 7-day-retention,
  daily-rotated, gzipped log under `/var/lib/ventd/logs/observations.log`.
  Locked schema in `docs/research/r-bundle/ventd-passive-observation-log-schema.md`.
  Consumed by future smart-mode patches (Layer B v0.5.7, Layer C v0.5.8,
  doctor v0.5.10) and v0.5.5 opportunistic probing.

- **New design system across every page** (#704). Dashboard, Devices,
  Sensors, Curves, Schedule, Calibration, Setup, Settings — all
  redesigned on a shared token system with consistent typography,
  spacing, and micro-interactions. Mobile responsive throughout.

### Bug fixes for the first-time-user path

Walkthrough validation on a fresh Ubuntu 24.04 install + Steam Deck +
MiniPC discovered an end-to-end chain of blockers that all ship in
v0.5.4:

- **Install: deferred service start** (#711) — the install script now
  waits for AppArmor + SELinux + udev rules to be in place before
  starting the daemon, and clears stale `/etc/apparmor.d/disable/<profile>`
  symlinks. Previously a fresh install could land in a denied-by-policy
  loop with no clear error.

- **First-boot wizard polish** —
  - AppArmor + listen + sd_notify so fresh installs actually work (#702)
  - Stop upper-casing the first-boot setup token (#712) — the token is
    case-sensitive; the input box was helpfully destroying it
  - CSP style-src violations + `/api/v1/profile/active` 405 polling (#713)
  - Drop inline `<script>` from `/` redirect helper (#714)
  - Remove dead `/logs` link from every sidebar (#715)
  - No-fan-host escape from `/calibration` (#717) — operators of
    sensor-only / monitor-only hosts can now exit the calibration page
    with a "Continue without fan control" button instead of being
    stuck on a stale "Preparing next fan…" view across daemon restart.
    Also adds the persistent `/var/lib/ventd/.setup-applied` marker
    so the wizard doesn't re-run on every restart.

- **Settings: change-password form** (#720) — the new form lives
  inline on the Settings page (instead of a broken modal), with three
  masked inputs, JSON body matching the API.

- **Daemon polish** —
  - `Progress.Applied` now honours the persistent marker (#723) — the
    JSON status no longer reports `applied: false` immediately after a
    daemon restart of an applied install.
  - `idle.StartupGate` preconditions go fork-free (#723) — no more
    `Got notification message from PID X` flood in `journalctl -u ventd`.
    The container detection and post-resume warmup checks now read
    `/proc/1/environ`, `/.dockerenv`, and `/run/systemd/units/...`
    directly instead of `exec.Command("systemd-detect-virt", ...)`.
