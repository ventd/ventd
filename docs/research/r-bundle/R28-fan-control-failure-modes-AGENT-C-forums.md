# R28 — Fan-Control Failure Modes (Agent C: Forum Mining)

**Bundle:** R28 — board-/chassis-specific fan-control quirks
**Agent scope:** Enthusiast Linux forums (Reddit, Phoronix Forums, Level1Techs, Tom's Hardware, ArchBBS, Manjaro, Linux Mint, Framework Community, Thinkpads.com, Lenovo Community, Acer Community, TrueNAS).
**Sister documents:** Agent A (GitHub issues), Agent B (distro / kernel / hwmon wikis). This document focuses on community-reported quirks where the fix is operator-actionable and field-confirmed by multiple users.
**Goal:** Build the lookup tables ventd's recovery classifier will encode as detect+auto-fix rules. The "Boards/Chips/Models affected" column is intentionally specific (e.g., "Gigabyte B650E AORUS ELITE X AX" rather than "Gigabyte AM5") because forum data shows the quirk-to-fix mapping is per-board, not per-family.

## Confidence rubric (forums)

- **low** — single forum post, single reporter, no independent confirmation; or thread closed unresolved.
- **medium** — multiple users in one thread confirm the same fix; or 2 distinct threads on different forums report the same fix for the same chip.
- **high** — fix appears across 3+ independent threads with multiple confirming users; consistent with mainstream wiki / Agent B finding.

## Anonymisation

Usernames are not reproduced; we record the issue, the fix, and the URL.

## Caveats

- `acpi_enforce_resources=lax` is widely used as a workaround but kernel maintainers warn it is not strictly safe (two drivers may bang the same I/O range). Where the OOT it87 driver supports it, prefer the per-driver `ignore_resource_conflict=1` parameter — narrower blast radius, same effect.
- A class of ASRock X670E and X870E boards has *no* known mainline fix as of kernel 6.5–6.11; sensors only work via force-loaded `nct6775` with a guessed `force_id`, and fan PWM control is partial or absent. Documented as upstream-track items at the bottom.
- Forum threads frequently die before resolution. Where this is the case, the row is marked low confidence with "no fix confirmed" in Notes.

## One-paragraph synthesis

The five highest-impact board family / quirk pairs in the forum data are: (1) **MSI MAG/MEG/PRO with NCT6687-R** (Tomahawk B650/B850/X870E, Z590, X570 — densely confirmed; fix is OOT `Fred78290/nct6687d` plus `fan_config=msi_alt1` and BIOS Smart Fan disabled); (2) **Gigabyte AORUS B650/X670/Z690 with ITE IT86xx** (`force_id=0x8689` for AM5, `0x8628` for older, plus `acpi_enforce_resources=lax` or per-driver `ignore_resource_conflict=1`); (3) **ASUS X570/B550 with NCT6798D** (people reach for asus-wmi-sensors but the fix is plain `nct6775` because no WMI exposed); (4) **Dell PowerEdge R-series / iDRAC 8/9** (`ipmitool raw 0x30 0x30` works only on iDRAC ≤3.30.30.30, removed by Dell on newer firmware); (5) **ThinkPad T/X/P series with thinkpad_acpi** (set `fan_control=1` in modprobe.d, plus on T14 Gen 3+ fall back to generic hwmon paths because `/proc/acpi/ibm/fan` is going away). **Trends over time:** Pre-Linux-6.3 X670/B650 threads were dominated by "no sensors at all" — kernel 6.3 mainlined large parts of asus-ec-sensors and nct6775 chip IDs, and post-6.3 threads shift to "sensors work, PWM partial" — ventd should advise kernel ≥6.3 as a first step on AM5. Late-2024/2025 X870E threads (ASRock Taichi X870E, MSI MAG X870E TOMAHAWK, Gigabyte X870E AORUS PRO) are repeating the X670E pattern but for fan PWM specifically — case fans visible in BIOS, invisible in Linux — suggesting another mainline driver lag wave. **Board families with NO known operator-actionable fix (UPSTREAM-TRACK):** ASRock X670E Pro RS / Steel Legend (no sensors at all); ThinkPad P15v Gen 2i (EC ignores level writes); Lenovo Yoga Slim 7i 13ITL5 (no driver, EC locked); Lenovo Yoga Slim 7 Pro 16ACH6 (ITE IT81201E EC, no driver); HP EliteBook 8xx / Pavilion (SMM-locked firmware); Framework Laptop 13 11th-gen on BIOS 3.22 with -274000°C poisoned readings; Gigabyte X670E AORUS MASTER (IT8689E rev 1, PWM no-op); Gigabyte X870E AORUS PRO (chained IT8792E missing); Dell iDRAC ≥3.34.34.34 (vendor removed feature); HP iLO 4 Gen 8/9 (BMC owns fans entirely). Several of these (Yoga Slim, P15v, EliteBook) reduce to "EC is locked, no public datasheet, no community RE work has matured" — ventd's classifier should detect the chassis and surface "fan control unavailable on this hardware" rather than thrash trying to find a working module.
