# ventd codebase map

A surface-by-surface navigation map of the whole repository: what each directory
is for, the key types and entry points to grep for, and the engineering-invariant
rule file that governs each subsystem. Read this first when you need to find
where something lives; open the cited files only when you need the full text.

- Module: `github.com/ventd/ventd` · Go **1.25.10** · `CGO_ENABLED=0` single static binary
- ~174k LOC Go · ~90 internal packages (top-level + sub) · single static binary
- Companion index of invariants: [`docs/rules/INDEX.md`](rules/INDEX.md) (regenerated by `go run ./tools/rule-index`)
- Narrative for newcomers: [`README.md`](../README.md) · API contract: [`docs/api.md`](api.md)

> **Keeping this current.** This file is a hand-maintained map, not generated.
> When you add a package, an `/api` route, a HAL backend, a CI workflow, or a
> rule file, update the matching section here. `tools/mapcheck` (run in
> `meta-lint.yml`) fails CI when a `cmd/` binary, a top-level `internal/`
> package, or a workflow is added without a reference here — so the structure
> can't silently drift. It does **not** check file counts (catalog/test/rule
> counts churn constantly via the catalog ingest), so this map avoids hard-coded
> numbers; run the relevant `ls` for current counts.

## 2. Binaries (`cmd/`)

| Binary | Role | Entry / notes |
|--------|------|---------------|
| `cmd/ventd/` | Main daemon **and** all operator subcommands | `main.go` `run()` (~2280 lines). Subcommand dispatch happens **before** `flag.Parse()` because subcommands own conflicting flag sets |
| `cmd/ventd-recover/` | Emergency one-shot: walk `/sys/class/hwmon/hwmon*/pwm<N>_enable`, write `1` (firmware-auto) | Fired by `ventd-recover.service` via `OnFailure=`. **Always exits 0** so the OnFailure chain can't loop. Alloc-light |
| `cmd/ventd-ipmi/` | Privilege-separated IPMI sidecar; sole holder of `/dev/ipmi0` (`CAP_SYS_RAWIO`) | Serves the daemon over a Unix socket using `internal/hal/ipmi/proto` (length-prefixed JSON) |
| `cmd/ventd-nvml-helper/` | SUID-root (`04755`) helper proxying whitelisted NVML *write* calls so the daemon can stay unprivileged | `/usr/local/sbin/ventd-nvml-helper`; stable exit codes 0–5; recursion guard |
| `cmd/ventd-soak/` | Read-only soak/field-validation harness; decodes persisted Layer-B coupling shards and prints convergence | Safe alongside production (`snapshot`/`watch` never write). `excite` needs `--enable-soak-excitation`. Not shipped by goreleaser |

**`ventd` subcommands** (all in `cmd/ventd/`): `diag bundle` / `diag export-observations` (`diag*.go`),
`doctor` (`doctor.go`), `preflight` (`preflight.go`, `preflightcheck.go`), `calibrate --acoustic`
(`calibrate_acoustic.go`), `import-sensors-conf` (`import_sensors_conf.go`), `power-profile`
(`power_profile.go`), `version`. Daemon-mode flags include `--setup`, `--probe-modules`,
`--rescan-hwmon`, `--recover`, `--list-fans-probe`, `--calibrate-probe`, `--preflight-check`,
the four experimental `--enable-*` flags, and `--strict-idle-gate`.

**Privileged vs sandboxed split:** `--probe-modules` / `--rescan-hwmon` / `--recover` are
privileged one-shots run by the installer **outside** the systemd sandbox. The long-running
daemon **never** modprobes — under `ProtectKernelModules=yes` those syscalls would EPERM.

## 3. The control tick and smart-mode data flow

This is the spine of the daemon. Read `cmd/ventd/main.go` (`run`, `runDaemonInternal`,
the `SmartModeBundle`) and `cmd/ventd/smart_builders.go` (`buildBlendFn`) alongside this.

**Bootstrap (`run()`):** subcommand dispatch → `config.LoadForStartup` (in-band retry absorbs
cold-boot udev ENOENT races, issue #103) → `calibrate.VerifyAllowStopGate` (refuse to start if a
fan has `allow_stop=true` without stall calibration, #600) → first-boot TLS self-sign + auth load
→ `probe.Probe` builds `[]*probe.ControllableChannel` → `registerHALBackends` + `hal.Enumerate`
→ `sysclass.Detect` → construct the `SmartModeBundle`. On first boot an inline background
**polarity auto-probe** runs under watchdog coverage; controllers refuse writes until polarity
persists (~115 s for 8 channels).

**SmartModeBundle (`main.go`)** bundles the learning runtimes derived in `run()`: signature
`Library`, Layer-B `coupling.Runtime`, Layer-C `marginal.Runtime`, Layer-A `layer_a.Estimator`,
`aggregator.Aggregator`, the `controller.BlendedController`, a `DecisionCache`, and the channel
slice. **Every builder returns nil in monitor-only mode** (no controllable channels), making all
downstream wiring a no-op.

**The per-fan reactive tick (`internal/controller/controller.go` `tick()`):**
1. Read `liveCfg` once (atomic pointer, swapped on SIGHUP).
2. Yield the tick if the channel is calibrating or panicked.
3. Read all sensors (flaky sensors isolated; sentinel values carried forward), EMA-smooth.
4. Evaluate the compiled curve (the PI curve is the only `StatefulCurve` — integral state carries across ticks).
5. **Clamp to `[MinPWM, MaxPWM]`** — the hard safety layer; the curve can never exceed operator bounds.
6. Run the blend hook (below) if wired.
7. Re-clamp, apply ramp-**down**-only hysteresis, then `polarity.WritePWM` (inversion-correct), then `ObsAppend`.
   PWM=0 is refused unless `MinPWM=0 AND AllowStop=true`.

**The predictive blend (`smart_builders.go` `buildBlendFn`), runs after the safety clamp,
before hysteresis:** pull Layer-B snapshot (`Coupling.Shard(chID).Read()`), Layer-C snapshot
(`Marginal.Shard(chID, signatureLabel).Read()`), Layer-A snapshot (`LayerA.Read(chID)`); derive
`confA/confB/confC`; call `Aggregator.Tick(...)` → `w_pred ∈ [0,1]`; call `Blended.Compute(...)`
→ `output = w_pred·predictive + (1-w_pred)·reactive`. Predictive is IMC-PI with magnitude gain
`K = |b_ii/(1-a)|` (signed-gain caused positive-feedback instability). Three refusal gates can
drop the predictive arm: instability guards, Path-A saturation (Layer-C ΔT < 2 °C), and the
acoustic-budget gate (R33 dBA proxy capped at the preset target). Result is re-clamped and cached
in `Decisions` for the `/api/smart/channels` UI.

**The confidence aggregation order is LOCKED** (R12 §Q3, `internal/confidence/aggregator`):
(1) per-layer drift decay → (2) MIN-collapse across A/B/C → (3) low-pass filter (τ=30s) →
(4) Lipschitz clamp (≤0.05/s). Plus a cold-start hard pin (`w_pred=0` for 5 min after Envelope-C)
and a global gate that bypasses the clamp to drop straight to safe.

**Background learning loops** (independent goroutines, ctx-scoped, one per daemon lifetime):
- `internal/probe/opportunistic` — idle-gated active probing of unvisited PWM bins; writes via `polarity.WritePWM`, feeds the observation log + the signguard sign-vote detector.
- `internal/observation` — the per-tick log that re-warms Layer-B/C on restart. **The controller→estimator bridge (`cmd/ventd/smart_obs_bridge.go`) is what feeds the RLS estimators**; its absence pre-v0.5.37 stranded them at θ=[0,0] (issue #1033).
- `internal/envelope` — one-shot Envelope-C/D thermal characterisation after the strict idle gate clears.

**Exit safety:** the `watchdog` (1.8 s restore budget, under systemd's 2 s `WatchdogSec`) restores
firmware-auto on exit/panic; `ventd-recover.service` is the `OnFailure=` backstop.

> Open issue worth knowing: convergence on 8+ channel boards (NCT6687D worked example, #1253) is
> the active smart-mode problem. The `wPredSystem` global gate is wired to real signals (R11 PR-1,
> `internal/confidence/gate` + `internal/massstall`); `driftFlags` remains stubbed at `[3]bool{}`
> (R11 PR-2 wires it via a per-layer EWMA-control-chart detector).

## 5. Package atlas — Hardware Abstraction Layer (HAL) and backends

### The HAL contract (`internal/hal/`)
- **`backend.go`** — `FanBackend` interface: `Enumerate(ctx)`, `Read(ch)` (must not mutate, sets `Reading.OK=false` on transient failure rather than erroring), `Write(ch, pwm uint8)` (forwards the exact 0–255 byte; controller owns clamping; owns mode transitions), `Restore(ch)` (idempotent), `Close()`, `Name()`. Channel caps bitset: `CapRead/CapWritePWM/CapWriteRPMTarget/CapRestore/CapWritePowerProfile`. Optional `PowerProfileBackend` (msi-ec shift_mode, etc.).
- **`registry.go`** — process-wide map guarded by RWMutex. `Register(name, b)` called **explicitly** from `cmd/ventd` (not `init()` — `go-conventions.md` forbids package init). `Enumerate(ctx)` fans out in sorted-name order and prefixes channel IDs `name:` (e.g. `hwmon:/sys/.../pwm1`). `Resolve(id)` splits on the first `:`.
- **Rule: `hal-contract.md`** (RULE-HAL-001..008). RULE-HAL-008: a repeat `Write` to an already-acquired channel must not re-issue the mode command (some firmware treats a repeat `pwm_enable=1` as a curve-timer reset → audible spike).

unconditionally once `Enumerate` returns a channel — no `--enable-X` opt-in. Safety = closed-set
enums + EC/register allowlists + watchdog restore + battery/container daemon refusal. The only
genuine experimental gate is AMD OverDrive (kernel taint).

**The hwdb catalog** lives under `internal/hwdb/catalog/` and is embedded: per-vendor board
files (`boards/<vendor>.yaml`, `board_profiles:` lists of `BoardCatalogEntry`), chip files
(`chips/`), and driver files (`drivers/`). The board catalog grows steadily via the catalog
ingest, so counts here are intentionally omitted; `ls internal/hwdb/catalog/boards` for the
current set. Matching is three-tier: Tier-1 exact DMI fingerprint (conf 0.9), Tier-1.5
chip-probe by hwmon name for mini-PCs whose BIOS leaves DMI as `"Default string"` (conf 0.85),
Tier-3 chip-family fallback (conf 0.6). Each board entry declares exactly one of
`dmi_fingerprint` / `dt_fingerprint` / `chip_probe`. **Navigate board data via
`profile_v1.go` + `profile_v1_1.go` + `matcher_v1.go`, not the legacy `schema.go`.**

## 7. Web UI and HTTP API

**Server (`internal/web/`, ~78 files).** Entry `web.New(...)` (`server.go`) builds one
`*http.ServeMux`, wires routes centrally, composes middleware, and launches `runHistorySampler`
+ `runScheduler`. `ListenAndServe(addr, cert, key)` binds; `Shutdown()` drains.

- **Routing** (`router.go` + `server.go`): static pages via `registerWebPage` (HTML no-cache, CSS/JS ETag+1h); every API route registered twice — `/api/<name>` **and** `/api/v1/<name>` — from one `apiRoute` table (`server.go`). Catch-all `/api/` → JSON 404.
- **Middleware** (`security.go`, `csrf.go`): global `securityHeaders` (strict CSP, no `unsafe-inline`) → `originCheck`; per-route `requireAuth` → `requireCSRF` → `requireMaxBody(1 MiB)`. Per-IP login rate limiter.
- **Health** (`health.go`): `ReadyState` (all-atomic). `/healthz` 200 once `SetHealthy`; `/readyz` 200 only if watchdog pinged AND a sensor read within 5 s. Both unauth.
- **Streaming** (`sse.go`, `setup_events_sse.go`): SSE only, **no websockets**. `/api/events` streams a `status` frame every 2 s; `/api/setup/events` streams the wizard activity feed (`?since=` resume).
- **TLS** (`selfsigned.go`, `http_redirect.go`): self-signed P-256 cert on first boot; a plaintext-on-TLS-port connection gets a 301 to `https://`.

**Auth (`internal/web/authpersist/`, `auth.go`, `csrf.go`).** Credentials live in `auth.json`
(bcrypt cost 12), **separate from config.yaml** so no config write can clobber the admin hash.
First-boot: `GET /login` 302→`/setup`; `POST /login` with `new_password` creates the admin
account (no setup token — eliminated v0.5.8.1). Sessions are in-memory (lost on restart).
Cookies: `ventd_session` (HttpOnly, SameSite=Strict) + `ventd_csrf` (readable by JS for the
synchroniser-token header injected in `web/shared/brand.js`).

**API endpoint groups** (full table with handler files: ask the source — `server.go` route table
~`389`–`495`). Auth = `requireAuth`+`requireCSRF`+body-cap unless noted:

- **Unauth probes:** `/healthz`, `/readyz`, `/api/ping`, `/api/auth/state`, `/api/version`.
- **Status/config/hardware:** `/api/status`, `/api/events` (SSE), `/api/config` (GET/PUT/PATCH), `/api/config/dryrun`, `/api/hardware`, `/api/hardware/inventory`, `/api/hardware/rescan`, `/api/history`, `/api/debug/hwmon`.
- **Profiles/schedule:** `/api/profile`, `/api/profile/active`, `/api/profile/schedule`, `/api/schedule/status`.
- **Panic:** `/api/panic`, `/api/panic/state`, `/api/panic/cancel`.
- **Calibration/probe:** `/api/calibrate/{start,status,results,abort,reset}`, `/api/probe/opportunistic/status`, `/api/detect-rpm`.
- **Setup/wizard:** `/api/setup/{status,events,start,apply,apply-monitor-only,reset,calibrate/abort,load-module}`, `/api/admin/factory-reset`, `/api/system/reboot`.
- **hwdiag one-click fixes (POST):** `/api/hwdiag` (GET report) + `/api/hwdiag/{install-kernel-headers,install-dkms,load-apparmor,mok-enroll,grub-cmdline-add,modprobe-options-write,reset-and-reinstall}`.
- **System introspection (GET):** `/api/system/{watchdog,recovery,security,diagnostics}`.

**Frontend (`web/`).** 13 vanilla pages (no framework/build): `dashboard`, `calibration`,
`curve-editor`, `devices`, `doctor`, `hardware`, `health`, `schedule`, `sensors`, `settings`,

## 8. Engineering invariants — the `docs/rules/` system

This is where the load-bearing invariants live (per project convention, **not** in `CLAUDE.md`).

- **`docs/rules/INDEX.md`** — the canonical map of every rule file. Read it first; open a specific rule only for full text. Regenerate with `go run ./tools/rule-index` (`--check` fails CI if stale).
- Each rule file states invariants as `## RULE-XXX` headings, many with a **`Bound:` line** naming a leaf subtest. `tools/rulelint` asserts that bound **leaf** subtest name exists via `t.Run(...)` — **`Parent/Leaf` form silently fails**, use the leaf name only.
- **`docs/rules-rationale/`** — long-form "why" behind rule families (smart-mode wiring, opportunistic soft-idle, hwmon runtime monitors).
- Rule families map 1:1 to the subsystems in §4–§7. The fast path from a symptom to the governing invariant is: grep the package name in `INDEX.md` → open the rule → find the bound test.

## 10. Deploy, packaging, and test infrastructure

**Deploy (`deploy/`).** systemd units carry the safety wiring:
- `ventd.service` — `Type=notify`, `Restart=on-failure`, **`WatchdogSec=2s`**, **`OnFailure=ventd-recover.service`**, `ExecStartPre=-/usr/local/sbin/ventd-wait-hwmon`. `User=root` with a heavy sandbox (`ProtectSystem=strict`, enumerated `ReadWritePaths`, `SystemCallFilter`, `MemoryDenyWriteExecute`). AppArmor profile shipped but **not** auto-attached.
- `ventd-recover.service` — `Type=oneshot`, resets every `pwm_enable=1` on failure.
- `ventd-ipmi.service` — sidecar, `User=ventd-ipmi`, `CAP_SYS_RAWIO`, `BindsTo` ventd.service, `ConditionPathExists=/dev/ipmi0`.
- `ventd-postreboot-verify.service` — opt-in post-reboot fan verification.
- udev: `90-ventd-hwmon.rules` (chgrp pwm files to `ventd`), `90-ventd-liquid.rules` (Corsair hidraw uaccess), `50-ventd-update.rules` (polkit for in-UI update). sysusers/tmpfiles drop-ins. SELinux `.te`/`.fc`.

**Packaging (`packaging/`):** `docker/` (alpine musl `nonvidia`), `aur/` (`ventd` + `ventd-bin`),
`nix/` (`flake.nix`, `nixos-module.nix`, drift check).

**Test infrastructure** (hundreds of `*_test.go` across the tree). Hardware fakes under `internal/testfixture/`:
