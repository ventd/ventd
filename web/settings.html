Smart mode

Behaviour of the autonomous fan-control learning loop. Disabling these stops ventd from learning new things about your fans; existing learned state is preserved.

Never actively probe after install
When off (default), ventd opportunistically writes a fan PWM value for ~30 seconds during genuinely-idle windows to fill gaps in the response curve. The first probe waits 24 hours after install. Disabling will leave coverage gaps in the low end of the curve.
Disable workload signature learning
When off (default), ventd hashes the names of currently-running processes locally to recognise repeating workloads (kernel build, game, browser) and learn workload-specific fan curves. Hashes never leave the machine; the per-install salt at /var/lib/ventd/.signature_salt protects against rainbow-table reversal of leaked diagnostic bundles.
Acoustic optimisation
When on (default), ventd refuses to ramp PWM further when the predicted temperature drop is less than 2 °C — the fans hold the quietest speed that still cools effectively. Toggle off if you want maximum-cooling-regardless-of-noise behaviour: ventd will follow the curve straight without the saturation gate.

Advanced

Destructive actions. Read carefully.

Reset to initial setup
Wipes the calibration KV namespace and the active config. The daemon restarts and the setup wizard opens again. Login credentials are kept.
Reset and reinstall driver
Removes any partially-installed OOT driver state — DKMS registration, .ko files in /lib/modules/<release>/extra/, modules-load.d entries, build dirs under /tmp/ventd-driver-* — so the next wizard run starts from a clean slate. Use after a botched first install or when switching catalog row.
Factory reset
Full state wipe — calibration, config, applied marker, AND login credentials. Lands on the password-set screen at /login as if this were a fresh install. The OOT driver under /lib/modules/<release>/extra/ is preserved (use "Reset driver" first if you want it gone too).
