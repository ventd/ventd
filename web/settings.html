Display

Theme and unit preferences. Stored per-browser; not synced to other clients.

Theme
Auto follows your operating system preference.
Temperature unit
Applied to every reading on Sensors, Devices, and the Dashboard.
Compact density
Tighter card padding and smaller sparklines. Useful at < 1280 px.
Reduce motion
Honours prefers-reduced-motion by default. Force-on disables propeller spin and pulse rings.

Profile schedules

Time grammar: HH:MM-HH:MM with optional weekday spec like mon-fri or sat,sun. Multiple windows may be space-separated. Leave blank for manual-only.

silent Silent fan profile · weeknights + late.
parsed · 2 windows · next ⇢ today 22:00
balanced Default daytime profile.
parsed · 2 windows · active now
performance Manual / event-driven only.
unrecognised token “weekdays” · expected mon-fri
calibrate Weekly fan-curve recalibration sweep.
parsed · 1 window · next ⇢ Sun 03:00
Schedules drive curve-set switches only. Per-fan manual overrides on Dashboard outrank the active profile until you clear them. The Schedule page shows the timeline and the priority order rules fire in.

Daemon

Live process settings. Changes here write to /etc/ventd/config.yaml and are applied on next poll — no restart required unless noted.

Listen address
Default 0.0.0.0:9999 binds all interfaces. Restrict to 127.0.0.1 if behind a reverse proxy.
Poll interval
How often the daemon reads sensors and recomputes curves. Lower values cost more CPU.
min 250ms
Session TTL
How long a browser stays signed in. Shorter values reduce risk if a workstation is left unlocked.
Auto-recover from crashes
Restart the daemon under systemd if it exits abnormally. Always-on in production builds.
Rescan hardware on boot
Re-runs hwmon + nvml discovery before applying curves. Slower boot, safer after kernel upgrades.

Security

ventd uses a single shared password; access is tied to an HTTP session cookie. Bind to 127.0.0.1 + a reverse proxy if you need multi-user auth.

Web password
Hashed with argon2id and stored in the config's web.password_hash field. Last changed 2025-09-04.

Danger zone

Operations that touch your config or restart the daemon. Read the fine print.

Restart daemon
Drops fans to motherboard defaults for ~2 s while the service comes back up. Curves resume once polling restarts.
Re-run setup wizard
Re-detects hardware and rebuilds curve defaults. Your existing curves are kept; mappings may change.
Reset to factory config
Removes /etc/ventd/config.yaml, every profile schedule, and your password. Setup wizard runs on next boot.
Wipe calibration data
Drops every fan's recorded stop/start PWM and min-stable RPM. Curves keep working but lose their floor estimates.
3 unsaved changes — listen address, session TTL, performance schedule grammar.
